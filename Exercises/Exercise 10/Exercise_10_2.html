Gegeben sei folgendes fehlerbehaftete JavaScript-Programm. Ergänzen Sie dieses um möglichst viele Typisierungen und übersetzen Sie es in ein TypeScript-Programm. 
const add = (x,y) => x+y;
const equals = (x,y) => x===y;

console.log( add(1,2) );
console.log( add(true, true ) );
console.log( add(true, false ) );
var x = y = 1;
console.log( add(equals(x,y), equals(y,x)) );

1) Was ist der Fehler in dem gegebenen JavaScript-Code? 

    Datentyp wird nicht berücksichtigt. Deshalb werden die booleschen Werte bei true=1 und false=0 addiert.

2) Geben Sie hier Ihren TypeScript-Code ein: 

    const add = (x:number, y:number) => x+y;
    const equals = (x:boolean, y:boolean) => x===y;
    console.log(add(1,2)); 
    console.log(add(true, true)); 
    console.log(add(true, false)); 
    var x = y = 1; 
    console.log(add(equals(x,y), equals(y,x)));

3) Welche Fehler meldet TypeScript? 

    1) console.log( add(true, true )); und console.log( add(true, false )); 

    Number wird erwartet und boolean wird gegeben. 

    2) var x = y = 1; 

    3) console.log( add(equals(x,y), equals(y,x)) ); 

    Number wird erwartet und boolean wird gegeben.

4) Geben Sie weitere Beispiele an, wie TypeScript bei der Fehlersuche helfen kann: 

    Durch ausführlische Kommentare im Code sind immer hilfreich für die Fehlersuche 
    const trueandfalse = (x: boolean) : 
        if (x === true) 
        {
            console.log('1');
        }

        if (x === false) {
            console.log('0');
        };
        trueandfalse(2);

        Die trueandfalse funktion wird nicht ausgeführt, weil sie NUR ein Boolean-wert erwartet.
